2

Unzip the file

Turn the downloaded zip into a folder.

›

Mac

  1. Open Finder.
  2. Go to Downloads.
  3. Double-click SoulAgentOS-v0.13.zip.
  4. A folder should appear.

Windows

  1. Open File Explorer.
  2. Go to Downloads.
  3. Right-click SoulAgentOS-v0.13.zip.
  4. Click Extract All.
After unzipping, you should see files like README.md, run.py, requirements.txt, and GITHUB_DOWNLOAD_INSTRUCTIONS.html.
3

Install or check Python

Recommended: Python 3.13.x, 64-bit, stable installer.

›

Recommended beginner choice

Download Python 3.13.x, 64-bit, stable installer. If python.org offers only a big “Download Python 3.14.x” button, that is also okay. Avoid anything labeled alpha, beta, rc, embeddable, or source.

Official Python downloads Mac installers Windows installers

Windows exact choice

  1. Open the Windows installers link.
  2. Look for Python 3.13.x under Stable Releases.
  3. Click Download Windows installer (64-bit).
  4. Run the installer.
  5. Important: check Add python.exe to PATH if you see that box.

Then check:

python --version

Mac exact choice

  1. Open the Mac installers link.
  2. Look for Python 3.13.x under Stable Releases.
  3. Click Download macOS installer.
  4. Open the downloaded .pkg installer and follow the prompts.

Then check:

python3 --version
4

Open Terminal / PowerShell in the project folder

This is the most important setup step.

›

Mac: easiest reliable method

  1. Open the unzipped SoulAgentOS-v0.13 folder in Finder.
  2. Open the Terminal app.
  3. Type cd with a space after it, but do not press Enter yet.
  4. Drag the SoulAgentOS-v0.13 folder from Finder into the Terminal window.
  5. Terminal will fill in the folder path.
  6. Press Enter.

You should now be “inside” the project folder. To check, type:

ls

You should see run.py and requirements.txt.

Windows: easiest reliable method

  1. Open the unzipped SoulAgentOS-v0.13 folder in File Explorer.
  2. Click once in the address bar at the top of the folder window.
  3. Type powershell.
  4. Press Enter.
  5. A PowerShell window should open already inside that folder.

To check, type:

dir

You should see run.py and requirements.txt.

Important: all commands below must be run from inside the folder that contains run.py.

Example asks for the agent

These are safe first prompts. Copy one into Bronson's First Agent after setup. The best prompts ask the agent to create an artifact, run it, inspect the output, and fix errors.

Program writing: tiny first test

Use this to confirm file-writing and stdout work.

Write a tiny Python program called hello_agent.py that prints hello from Bronson's First Agent. Run it and show me stdout.

Program writing: build a little app

Good for seeing it create something visual in the workspace.

Create a single-file HTML toy called bouncing_ball.html. It should show a ball bouncing with gravity, pause/play, and a reset button. Save it in the workspace. Then tell me exactly which file to open.

Debugging: run and repair

Good for testing the Lab Bench and auto-fix loop.

Write a Python script that calculates the first 20 Fibonacci numbers, saves them to fibonacci.txt, runs the script, checks stdout/stderr, and fixes any error before giving the final answer.

Data/chart task

Good for seeing it make a file and a chart.

Create a small fake CSV of 14 days of step counts, write a Python script to analyze it, make a simple matplotlib chart, save the chart as steps_chart.png, run the script, and show me stdout/stderr.

Research online

Use only if your version/provider has online search or a research tool configured.

Research online: find 5 recent reputable sources about independent AI agents and autonomous businesses. Make a short report with links, dates, and a one-paragraph summary. Save it as ai_agents_research.md in the workspace. Do not invent sources.

Research with caution

For current facts: demand citations and dates.

Research online: what are the latest stable Python versions for Mac and Windows? Use official Python.org sources only. Give me the recommendation for a non-technical user and include dates/links. Save the answer as python_version_note.md.

Knowledge graph / memory

Good for seeing whether it records useful facts.

Add this to your typed knowledge graph: Bronson's First Agent is a local experimental agent lab. Then retrieve related facts about tools, lab bench, and biopsy, and explain what you found.

Biopsy test

Good for learning the black-box recorder.

Intentionally create a tiny Python script with one obvious bug, run it, inspect the error, fix it, rerun it, and then tell me what would be useful to look for in Biopsy if this had failed.

Avoid these asks

  • Do not ask it to email people automatically.
  • Do not ask it to scrape private contact information.
  • Do not ask it to make financial, legal, medical, or public-outreach decisions without human review.
  • Do not trust online research unless it gives sources and dates.

Heartbeat demo: set it to 1 minute

For a demo, set the heartbeat / Goal Pulse interval to 1 minute or 60 seconds so you can see it act quickly. Use harmless goals only.

How to set it

  1. Open Bronson's First Agent in your browser.
  2. Find the Heartbeat, Goal Pulse, or similar panel.
  3. Set interval to 1 minute or 60 seconds.
  4. Use a harmless local goal from the list below.
  5. Start the heartbeat.
  6. Watch the trace/log. Stop it after a few cycles.

Important demo rule

One-minute heartbeat is for seeing the machine move. Do not give it external powers. No email sending, no social posting, no purchases, no public outreach, no destructive file cleanup.

Heartbeat demo: lab notebook

Heartbeat goal: every pulse, add one short entry to heartbeat_demo_log.md saying the time, what you inspected, and the next tiny safe step. Do not contact anyone. Do not use the internet. Stop after 5 useful entries.

Heartbeat demo: improve one file

Heartbeat goal: improve README_notes.md one tiny section at a time. On each pulse, make at most one small edit, record what changed, and stop after 5 pulses.

Heartbeat demo: skill practice

Heartbeat goal: practice using the Skill Finder safely. On each pulse, choose one harmless built-in skill, explain when it helps, and append the note to skill_practice.md. Stop after 5 pulses.

Heartbeat demo: local research only

Heartbeat goal: inspect the local project files only. On each pulse, summarize one file's purpose in project_map.md. Do not use the internet. Stop after mapping 5 files.

If stuck, paste or upload a screenshot to GPT

This is the easiest support method. Send GPT a screenshot of the error, folder, Terminal, PowerShell, or browser page.

Mac screenshot

  1. Press Command + Shift + 4.
  2. Drag around the error or window.
  3. A screenshot will appear on your Desktop.
  4. Upload or drag that screenshot into ChatGPT.

Windows screenshot

  1. Press Windows + Shift + S.
  2. Drag around the error or window.
  3. The screenshot is copied.
  4. Paste it into ChatGPT with Ctrl + V.

In ChatGPT

  1. Click the + button near the message box.
  2. Choose Add photos & files, or drag/paste the screenshot.
  3. Say: “Tell me exactly what to click or type next.”

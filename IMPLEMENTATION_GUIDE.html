1. What this is

SoulAgentOS is a local agent laboratory. It is not a black-box autonomous system. It is a controlled loop where a language model can choose from a fixed set of local tools, write files only inside workspace/, run Python files there, and show you exactly what happened.

Observe context
Choose one action
Run tool
Capture result
Continue or final
Biopsy if broken
Core idea
The agent should be useful because it can make and run artifacts, not because it sounds confident.
Safety boundary
No shell skill. Local file access is restricted to workspace/. Python runs are low-stakes lab execution, not a hardened sandbox.
Killer feature
Biopsy exports the raw prompts, model outputs, tool calls, stdout/stderr, goals, memory, and workspace state so a failure can be diagnosed.

4. First test: prove the agent loop works

Start with a tiny task that forces a file write, program run, and visible output.

Recommended test prompt

Write a Python program in workspace that prints the first 20 prime numbers. Run it. If it fails, fix it and run again. Then tell me the stdout.

Expected behavior:

  1. Live trace shows the model deciding to write a file.
  2. The file appears in the workspace selector.
  3. The app automatically runs the new .py file.
  4. Program Output shows stdout/stderr.
  5. The final answer references real run output, not just confidence.
Red flag: If the final says “I wrote and ran it,” but workspace/ is empty or Program Runs is empty, use Biopsy. That is exactly the failure v8 was designed to expose.

5. Program Lab Bench

The Lab Bench is the center of the app for coding tasks. It should feel like a small IDE plus a specimen tray.

1Select file
Choose any file inside workspace/. Python files are runnable.
2Edit code
Inspect or edit the code in the viewer/editor.
3Run Program
Execute with the python_run skill. No shell.
4Read stdout/stderr
Output is saved to SQLite and shown in Program Output.
5Run + Auto-Fix
The agent reads the failure, patches the file, and reruns up to max steps.

Auto-fix loop

run selected .py
if returncode == 0:
    show stdout and stop
else:
    send code + stderr to agent
    agent patches file_write
    run again
    repeat until success or max steps
Security note: The runner is deliberately limited but not a hardened sandbox. Do not use it to run untrusted code from strangers. It is for your local low-stakes lab work.

6. Goal Pulse

Goal Pulse is the long-horizon loop. It is visible, browser-driven, and should move user work forward before touching agent housekeeping.

ModeBest useExpected behavior
workDefaultPick highest-priority open user goal/todo and produce an artifact if possible.
advanceStepwise project progressMove one concrete todo forward.
cleanupMemory/workspace hygieneOnly after checking whether user work is waiting.
reflectStatus checkSummarize state without unnecessary changes.

Use Pulse Once Now for testing. Use Start Pulse to repeat while the browser tab is open. The pulse is not a hidden daemon.

Good pulse test

Add a high-priority todo: write a Python script that finds the first 20 four-digit primes in pi and e, run it, and save the output. Then pulse once.

7. Biopsy workflow

Biopsy is the debug contract. When the app fails, you do not describe the ghost. You export the specimen.

  1. Click Biopsy or Copy Biopsy.
  2. Paste the JSON into ChatGPT.
  3. Ask: “Diagnose the failure mode and patch the code.”

Biopsy includes

  • App version and browser timestamp.
  • Settings without API key.
  • Raw model output.
  • Parsed JSON or parsed function-call blocks.
  • Tool calls and tool results.
  • Program stdout/stderr.
  • Workspace file list.
  • Soul.md, goals, todos, KG, notes, and recent pulse history.
Important: The API key field is redacted/not exported. Still, Biopsy can include local notes and prompts. Treat it as a diagnostic bundle, not a public paste.

8. Adding a custom skill

Skills are registered in tools.py. A skill is just a Python function plus a JSON schema describing its arguments.

Minimal skill example

def skill_reverse_text(text: str):
    return {"reversed": text[::-1]}

TOOL_REGISTRY["reverse_text"] = Tool(
    "reverse_text",
    "Reverse a string.",
    {
        "type": "object",
        "properties": {"text": {"type": "string"}},
        "required": ["text"],
    },
    skill_reverse_text,
)
  1. Open tools.py.
  2. Add the function.
  3. Add the Tool(...) entry.
  4. Restart Flask.
  5. Ask the agent to use the new skill.
Approval gates: Any future skill that sends email, deletes files, spends money, calls APIs with private data, or changes external systems should require a human approval screen before execution.

9. Shipping checklist

Clean secrets
  • Do not ship .env.local.
  • Do not ship real API keys.
  • Keep .env.example.
Clean data
  • Ship data/.keep, not a personal DB.
  • Ship workspace/.keep, not private workspace files.
First-run UX
  • Name field visible.
  • Agent name visible.
  • Model selector visible.
  • Test prompt included.
Safety banner
  • Local experimental agent.
  • Workspace-only files.
  • Python runner is not a sandbox.
  • Biopsy may include local notes.

Suggested repository description

SoulAgentOS is a local Flask-based agent laboratory with API-key model access, Soul.md, SQLite memory, goals/todos, knowledge graph triples, a visible Goal Pulse, workspace-caged file tools, Python stdout/stderr capture, auto-fix loop, and Biopsy export for debugging failures.

Suggested README warning

This is an experimental local agent lab. It can write and run Python files inside workspace/. It is not a hardened sandbox. Do not run untrusted code. Do not paste secrets except API keys into the intended API key field. Use Biopsy to debug failures; review Biopsy contents before sharing publicly.
